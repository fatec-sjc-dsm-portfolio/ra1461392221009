DESENVOLVIMENTO DE SISTEMA DE RECUPERAÇÃO E AUTENTICAÇÃO DE CONSULTA PARA MELHORIA DE EXPERIÊNCIA DE USUÁRIO

Sistema web para recuperação e autenticação de consultas, permitindo a geração de relatórios confiáveis e otimizando a experiência do usuário

MIDTI - Monitoria de Iniciação em Desenvolvimento Tecnológico e Inovação

Professor orientador: FABRÍCIO GALENDE MARQUES DE CARVALHO

Sobre o Projeto

O projeto consistiu no desenvolvimento de uma aplicação web para gerenciamento de relatórios de queimadas no Brasil, com funcionalidades de busca, armazenamento de parâmetros, geração de relatórios em PDF e autenticação de dados. A aplicação foi construída com um CRUD completo, utilizando Node.js, Express, MongoDB e PDFKit, com foco na experiência do usuário e na confiabilidade dos relatórios.

Tecnologias Utilizadas

  • JavaScript: Linguagem principal para desenvolvimento web.
  • Node.js: Plataforma para execução de JavaScript no lado do servidor.
  • Express: Framework web para Node.js, usado para gerenciar rotas e requisições.
  • MongoDB: Banco de dados NoSQL orientado a documentos.
  • Mongoose: Biblioteca para modelagem de dados com MongoDB.
  • PDFKit: Biblioteca JavaScript para geração de arquivos PDF.
  • EJS: Template engine para renderização de HTML dinâmico.
  • Faker: Biblioteca para geração de dados fictícios.

Contribuições

  • Implementação de um sistema CRUD completo.
  • Integração com MongoDB usando Mongoose para gerenciamento de dados.
  • Desenvolvimento de rotas assíncronas com Express para buscas dinâmicas.
  • Geração de relatórios em PDF com a biblioteca PDFKit.
  • Criação de dados fictícios com a biblioteca Faker para testes de escalabilidade.

Impacto:Redução do tempo necessário para gerar relatórios autenticados, garantindo maior confiabilidade e facilidade de acesso a informações para tomadas de decisão.

SISTEMA PARA MONITORAMENTO DE UTILIZAÇÃO DE APLICAÇÕES PARA A INTERNET VOLTADO AO PROJETO DE UX

Sistema front-end para captura e análise de interações de usuários em aplicações web, com foco em melhorar a experiência do usuário (UX) através de dados quantitativos

PIBITI - Programa Institucional de Bolsas de Iniciação em Desenvolvimento Tecnológico e Inovação

Professor orientador: FABRÍCIO GALENDE MARQUES DE CARVALHO

Sobre o Projeto

Desenvolvido no Instituto Nacional de Pesquisas Espaciais (INPE), este projeto teve como objetivo criar um sistema de monitoramento de interações do usuário em aplicações web, com foco especial em interfaces geoespaciais. Utilizando uma abordagem metodológica que integra aspectos qualitativos, quantitativos e exploratórios, o sistema captura eventos como cliques, carregamentos de página e interações com mapas, armazenando esses dados para análise de padrões de uso e identificação de áreas de melhoria na experiência do usuário (UX). O projeto está sendo implementado no portal Queimadas do INPE, contribuindo para uma melhor compreensão de como os usuários interagem com informações geoespaciais.

Tecnologias Utilizadas

  • JavaScript: Linguagem principal para desenvolvimento front-end e back-end (Node.js).
  • Leaflet: Biblioteca open-source para mapas interativos e visualização de dados geoespaciais.
  • jQuery e AJAX: Para manipulação do DOM e comunicação assíncrona com o servidor.
  • React: Biblioteca para construção do dashboard de visualização de métricas.
  • Node.js: Ambiente de execução para o serviço de persistência de dados.
  • TypeScript e TypeORM: Usados no back-end para tipagem estática e acesso ao banco de dados.
  • MariaDB: Sistema de gerenciamento de banco de dados relacional para armazenamento dos eventos.
  • Selenium: Para automação de testes de interação do usuário.
  • Docker: Para conteinerização da aplicação e garantia de consistência entre ambientes.

Contribuições

  • Desenvolvimento de componentes front-end para captura de eventos de interação do usuário (cliques, carregamentos, interações com mapas).
  • Implementação de sistema de controle de sessão para análise precisa das métricas de UX.
  • Criação de dashboards interativos para visualização de métricas de uso das aplicações.
  • Integração com back-end para armazenamento e análise dos dados coletados.
  • Desenvolvimento de componentes específicos para captura de eventos em mapas e interfaces geoespaciais.

Impacto: O sistema proporcionou uma compreensão mais profunda de como os usuários interagem com as aplicações do INPE, identificando padrões de uso e áreas de melhoria. A implementação no portal Queimadas permitirá melhorias contínuas na experiência do usuário, com decisões baseadas em dados concretos de interação.
