Octoplus - Internet Banking

Um sistema de Internet Banking desenvolvido para monitorar e gerenciar operações bancárias, minimizando interações presenciais.

1º Semestre • 2/2022

Parceiro Acadêmico: Faculdade de Tecnologia de São José dos Campos

Octoplus - Repositório do Projeto

Sobre o Projeto

O projeto em foco concentra-se no desenvolvimento de um sistema de Internet Banking, com o propósito de monitorar e gerenciar operações bancárias, visando minimizar as interações presenciais.

Nossa abordagem utilizou o modelo MVC, concentrando-se na construção de uma aplicação em Flask para resolver os desafios propostos pelo cliente. Durante o desenvolvimento, aplicamos conhecimentos adquiridos em disciplinas do primeiro semestre, complementados por pesquisas extensivas fora do ambiente acadêmico.

Nossa equipe, denominada Octoplus, adotou metodologias ágeis para a entrega de funcionalidades essenciais e para o desenvolvimento de uma interface administrativa intuitiva. Os principais atores do sistema são o cliente, o gerente de agência e o gerente geral.

Tecnologias Utilizadas

  • Figma: para o desenvolvimento e prototipação das wireframes.
  • HTML, CSS e JS: para o desenvolvimento da página web e estilização.
  • Python e Flask: para a construção da lógica e requisitos solicitados pelo nosso cliente.
  • MySQL: como SGBD do nosso projeto.
  • AWS: para colocar nosso projeto rodando em nuvem.

Contribuições Pessoais

Como Product Owner (PO), minha principal responsabilidade foi garantir a compreensão precisa das necessidades do cliente e traduzi-las para o time de desenvolvimento. Atuei como facilitador, definindo requisitos, priorizando tarefas e assegurando que o produto final atendesse às expectativas.

Octagro - ERP de Inspeção de Entrada

Um sistema para controle de recebimento de grãos, desenvolvido em parceria com a Jaia Software.

2º Semestre • 1/2023

Parceiro Acadêmico: Jaia Software

Octagro - Repositório do Projeto

Problemática do Projeto

Em parceria com a empresa Jaia Software, nosso objetivo primordial foi aprimorar o controle do recebimento de grãos na agroindústria. Identificamos a necessidade de superar os obstáculos presentes nesse processo, que carece de critérios rigorosos de aprovação e relatórios abrangentes para otimizar a eficiência operacional.

O principal foco foi mitigar as dificuldades associadas à ausência de um sistema centralizado para inspeção de entrada. Essa lacuna impacta a segurança, qualidade, manutenção e gerenciamento de riscos. Para isso, nos comprometemos a desenvolver um sistema dedicado à "Inspeção de Entrada", visando não apenas um desafio técnico, mas também a compreensão abrangente do processo de recebimento de grãos.

Nossa solução foi elaborada com base nas necessidades específicas do processo, incorporando funcionalidades essenciais para uma verificação minuciosa e rigorosa dos grãos no momento de sua entrada na empresa. Isso envolveu diferentes etapas de validação realizadas por usuários distintos, visando garantir precisão e eficiência no processo.

No âmbito do sistema desenvolvido, criamos distintas funções para os usuários envolvidos. O Recebedor foi designado para efetuar a entrada dos grãos, enquanto o Analista verificava a documentação e procedimentos realizados de maneira precisa. O Aprovador, por sua vez, finalizava a entrada e o Administrador possuía permissões amplas, podendo executar todas as ações anteriores e também criar novos usuários para o ERP.

Tecnologias Utilizadas

  • Figma: para o desenvolvimento e prototipação das wireframes.
  • React e Module CSS: para o desenvolvimento da página web e estilização.
  • JavaScript, Node e Express: para a construção da lógica e requisitos solicitados pelo nosso cliente.
  • MySQL: como SGBD do nosso projeto.

Contribuições Pessoais

Concentrei meus esforços no desenvolvimento do protótipo inicial e na implementação do sistema dedicado ao Administrador e Analista. Minha principal área de atuação foi no front-end, utilizando React para criar componentes reutilizáveis e uma interface intuitiva.

Lições Aprendidas

Trabalhei na criação de um protótipo navegável utilizando o Figma, além de liderar o design e o desenvolvimento front-end. Aprofundei meu conhecimento em React e JavaScript, garantindo uma experiência de usuário coesa e eficiente.

Ninetech - Software de Gestão de Times de Projetos Complexos

Um sistema para simplificar a gestão de equipes multidisciplinares em projetos complexos, desenvolvido em parceria com a SIATT.

3º Semestre • 2/2023

Parceiro Acadêmico: SIATT

Ninetech - Repositório do Projeto

Sobre o Projeto

Esta iniciativa consiste no desenvolvimento de um software que simplifica a gestão de equipes multidisciplinares em projetos complexos. Criado para atender a empresa parceira Siatt, atuante no setor armamentista, o sistema se destaca pelo uso de metodologias consolidadas, como o IPDT (Integrated Product Development Team) e o WBS (Work Breakdown Structure). Estas metodologias estruturam equipes para o desenvolvimento de projetos complexos. O Sistema de Gerenciamento de Times para Projetos Complexos oferece soluções abrangentes, permitindo tanto o gerenciamento macro quanto micro dos projetos estratégicos da Siatt, proporcionando uma flexibilidade essencial para lidar com a complexidade inerente a esses empreendimentos.

Tecnologias Utilizadas

  • Figma: para o desenvolvimento e prototipação das wireframes.
  • React e Sass: para o desenvolvimento da página web e estilização.
  • Java, com Spring Boot: para a construção da lógica e requisitos solicitados pelo nosso cliente.
  • MySQL: como SGBD do nosso projeto.

Contribuições Pessoais

Durante o projeto, desempenhei o papel crucial de desenvolvedor FrontEnd, assumindo a responsabilidade pela criação e implementação das diversas telas do sistema. Meu foco foi não apenas na concepção visual, mas também na integração cuidadosa entre o FrontEnd e o BackEnd, assegurando uma experiência de usuário coesa e funcional.

Ao longo das sprints, liderei a criação de telas, resolvendo erros iniciais, realizando a integração entre camadas, além de implementar validações e gerenciar os usuários. Utilizei o React como principal estrutura do projeto e apliquei técnicas de estilização com CSS, garantindo uma interface responsiva e atraente para os usuários.

Lições Aprendidas

Durante o processo, enfrentamos desafios significativos que proporcionaram aprendizados valiosos. A falta de comunicação resultou em atrasos nas entregas, agravados pela saída de peças fundamentais da equipe, o que exigiu uma reestruturação para manter o projeto em andamento.

A demora inicial no entendimento do projeto destacou a importância da clareza nas expectativas e na documentação para uma execução eficaz. A autonomia concedida para criar páginas e corrigir erros foi uma oportunidade valiosa, permitindo o desenvolvimento de habilidades de resolução de problemas e inovação.

Esses desafios fortaleceram a equipe, enfatizando a importância da comunicação contínua, do planejamento e da adaptação ágil diante de obstáculos inesperados.

LW Solution - Software para Estações Meteorológicas

Desenvolvimento de estações meteorológicas de baixo custo para coleta e transmissão de dados ambientais.

4º Semestre • 1/2024

Parceiro Acadêmico: TECSUS

LW Solution - Repositório do Projeto

Problemática do Projeto

O foco deste projeto foi o desenvolvimento de estações meteorológicas de baixo custo capazes de coletar dados precisos sobre condições ambientais, como direção e velocidade do vento, índice pluviométrico, umidade, temperatura e pressão atmosférica, transmitindo-os de forma confiável para um servidor central. Os dados capturados foram processados e exibidos em um portal online, que além de fornecer informações em tempo real sobre o clima local, também desempenhou um papel educativo ao explicar os conceitos matemáticos por trás dos cálculos dos parâmetros meteorológicos e destacar a importância do monitoramento ambiental na prevenção de desastres naturais.

Tecnologias Utilizadas

  • Figma: para o desenvolvimento e prototipação das wireframes.
  • React, MaterialUI e Bootstrap: para o desenvolvimento da página web e estilização.
  • Node.js, Typescript e Javascript: para a construção da lógica e dos requisitos solicitados pelo cliente.
  • MySQL, PostgreSQL e Redis: como sistemas de gerenciamento de banco de dados (SGBD) do projeto.
  • Docker: para a conteinerização e gestão dos ambientes de desenvolvimento e produção.
  • Python: utilizado em scripts e automações para processamento de dados e outras necessidades específicas.

Contribuições Pessoais

Neste projeto, assumi a responsabilidade pelo desenvolvimento do front-end, onde criei dashboards e tabelas para a exibição de dados, além de ser responsável pelo layout completo da aplicação. Além das minhas funções técnicas, também desempenhei o papel de Product Owner (PO), combinando as responsabilidades de desenvolvimento e gestão de produto.

Apliquei a cultura DevOps e práticas de CI/CD, onde fui responsável pela rastreabilidade das atividades DevOps. Utilizando o GitHub Projects, organizei o backlog e as tasks do projeto, garantindo que toda a equipe de desenvolvimento tivesse acesso fácil e transparente às informações e ao progresso do projeto. Também implementei padrões de branches para manter a consistência e a qualidade do código.

Lições Aprendidas

Desempenhando simultaneamente as funções de gestor e desenvolvedor, aprendi a importância de equilibrar responsabilidades técnicas e de liderança, especialmente em um ambiente de equipe reduzida. A sobrecarga foi um desafio, exacerbada pela produção limitada de alguns membros da equipe, o que ressaltou a necessidade de uma gestão de recursos mais eficaz. A implementação de uma cultura DevOps e de práticas de CI/CD também se mostrou difícil, exigindo mais tempo e esforço do que inicialmente previsto. No entanto, essas dificuldades foram acompanhadas por valiosas lições sobre a importância da comunicação clara e da definição de processos. Por outro lado, a adoção de micro-serviços e a conteinerização com Docker demonstraram ser extremamente práticas e eficazes, simplificando a gestão de ambientes e contribuindo para a escalabilidade e eficiência do projeto.

noCloud - Mapeamento Automático de Nuvens e Sombras

Aplicação mobile e serviço em nuvem para mapeamento automático de nuvens e sombras em imagens do satélite CBERS4A/WPM, utilizando Deep Learning.

5º Semestre • 2/2024

Parceiro Acadêmico: Visiona

noCloud - Repositório do Projeto

Problemática do Projeto

Em parceria com a empresa Visiona, o projeto visa desenvolver uma aplicação mobile integrada à computação em nuvem, com foco no mapeamento automático de nuvens e sombras de nuvens em imagens capturadas pelo sensor WPM, a bordo do satélite CBERS4A. Essa aplicação tem como objetivo fornecer máscaras de nuvens de alta precisão para as imagens adquiridas, facilitando o monitoramento e a análise de diversas áreas de interesse no Brasil.

A ausência de máscaras de nuvens nas imagens do satélite CBERS4A/WPM limita a capacidade de análise de grandes volumes de imagens em áreas extensas, especialmente em regiões tropicais com alta cobertura de nuvens. Quando realizado manualmente, esse processo é custoso e demorado, dificultando a automação de procedimentos em aplicações que exigem alta precisão e eficiência.

O projeto propõe o desenvolvimento de um serviço e uma aplicação mobile em nuvem que automatiza o mapeamento de nuvens e sombras de nuvens em imagens do satélite CBERS4A/WPM, utilizando modelos de Deep Learning. A solução permitirá a definição de área e período de interesse, acesso às imagens correspondentes, mapeamento automático de nuvens e sombras, visualização dos resultados em uma interface de mapas, e download das máscaras em formato vetorial. Isso tornará o processo de análise mais eficiente, preciso e acessível para os usuários.

Tecnologias Utilizadas

  • Python: para desenvolvimento dos modelos de Deep Learning e processamento das imagens.
  • Firebase: para autenticação, armazenamento e gerenciamento de dados em nuvem.
  • PostgreSQL: como banco de dados relacional para armazenamento estruturado das informações.
  • Flutter: para o desenvolvimento da aplicação mobile multiplataforma.
  • TypeScript: para garantir tipagem e robustez no desenvolvimento do front-end e scripts.
  • JavaScript: para funcionalidades dinâmicas e integração entre sistemas.
  • Iolo: para automação de fluxos e integração entre serviços.
  • Roboflow: para gerenciamento e treinamento dos datasets de visão computacional.
  • Figma: para prototipação e design das interfaces da aplicação.

Contribuições Pessoais

Atuei como Product Owner (PO) do projeto, sendo responsável pelo contato direto com o cliente, negociação de entregas e alinhamento das expectativas. Propus soluções inovadoras, como a possibilidade de importação de arquivos no formato PNG e a integração com a API do INPE, permitindo o processamento direto das imagens do satélite. Fui responsável por toda a prototipação do sistema, além de desenvolver integralmente o front-end mobile e web utilizando Flutter, tecnologia na qual possuo grande domínio. Também destaquei-me na gestão do projeto, conduzindo a equipe para a conclusão das etapas dentro do prazo estabelecido, garantindo solidez, alta produtividade e qualidade nas entregas.

Lições Aprendidas

Este projeto foi fundamental para aprimorar minhas habilidades de organização e comunicação, tanto com o cliente quanto com a equipe e demais stakeholders. Aprendi a importância de manter um alinhamento constante entre todos os envolvidos, garantindo que as expectativas fossem atendidas e que as entregas ocorressem dentro dos prazos estabelecidos. Além disso, tive a oportunidade de aprofundar meus conhecimentos em Flutter, uma tecnologia robusta que ampliou significativamente minhas possibilidades profissionais e abriu portas para novas oportunidades no mercado. O desafio de gerenciar o projeto do início ao fim me proporcionou uma visão mais estratégica e madura sobre liderança, colaboração e tomada de decisão em ambientes de alta complexidade.
